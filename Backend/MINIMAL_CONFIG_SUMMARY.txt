📊 CONFIGURATION BREAKDOWN
═══════════════════════════════════════════════════════════════════════════════
application.properties (In each service):
  └─ 2-3 lines only (name + config-import)
application.yml (In config-server):
  └─ 2 lines only (name + profile)
Config Server YAML files (In GitHub config repo):
  └─ Complete service configuration
     - Server port
     - Database connection
     - Eureka configuration
     - Logging settings
     - Service-specific configs

🚀 STARTUP FLOW
═══════════════════════════════════════════════════════════════════════════════
1. Start discoveryServer
   └─ Loads application.properties (minimal)
   └─ Fetches discoveryServer.yml from config-server
2. Start config-server
   └─ Loads application.yml (minimal)
   └─ Loads config-server.yml from its own config folder
   └─ Serves all YAML files to other services
3. Start other services
   └─ Load application.properties (minimal)
   └─ Contact config-server at http://localhost:8888
   └─ Fetch {serviceName}.yml configuration
   └─ Register with Eureka
